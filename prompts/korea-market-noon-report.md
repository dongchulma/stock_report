# 한국 증시 오전 장중 리포트 (매일 12:00 KST)

## 목적
정오 기준 한국 증시 장중 흐름을 정리하는 리포트를 작성한다.

## 공통 규칙 (반드시 지킬 것)
1. 리포트는 반드시 한국어로 작성한다.
2. 리포트에 들어가는 모든 숫자 지표(지수, 종목 가격, 환율, 원자재, 금리 등)는 전일 대비 변동률(%)을 함께 표기한다. 변동률을 구할 수 없으면 그 사실을 명시한다.
3. 각 지표는 숫자만 나열하지 않고, 대화체로 짧은 설명을 덧붙인다 (예: "코스피가 어제보다 1.5% 올랐네요, 반도체주가 끌어준 덕분이에요.").

## 포함할 내용
1. 오전 장 종합 흐름 요약 + 핵심 테마
2. 한국 주요 지수 (장중 기준): KOSPI, KOSDAQ, KOSPI200, 현재가/장중 고점/등락률
3. 미국 국채금리: 2년물, 10년물, 30년물 + 스프레드 (전일 기준)
4. 반도체 관련: DRAM/NAND/HBM 동향, 한국 반도체주 장중 동향
5. 하이퍼스케일러 반도체 Capex 동향
6. 환율 & 원자재: USD/KRW(장중 범위), WTI, Gold
7. 오전 장 특이사항: 외국인/기관/개인 매매 동향, 주요 테마 3개 내외

데이터는 WebSearch/WebFetch로 최신 정보를 조사해서 채운다. 장중 실시간 수치를 구할 수 없으면 가장 최근 확인 가능한 값을 쓰고 기준 시점을 표기한다.

## 포맷
- 한국어, 텍스트 구분선(━━━)과 화살표(▶)/경고(⚠️) 기호 위주
- 텔레그램 메시지로 읽기 좋게 작성한다
- 마지막에 투자 책임 안내 문구를 남긴다

## 실행 절차
0. **휴장일 확인 (가장 먼저, 반드시 자동으로 판단할 것)**: 오늘이 한국 증시 휴장일(주말 또는 공휴일)인지 WebSearch로 확인한다.
   이 작업은 headless로 실행되어 사용자에게 질문할 수 없다 — 휴장 여부를 스스로 판단하고, 절대 사용자에게 되묻지 않는다.
   휴장이면 리포트 작성과 텔레그램 전송을 모두 건너뛰고, 곧바로 아래를 실행한 뒤 종료한다:
   ```
   python3 scripts/update_status.py korea-noon <YYYY-MM-DD> skipped_holiday
   ```
   개장일이면 아래 1~3단계를 정상적으로 진행한다.
1. 리포트를 로컬 파일(예: `/tmp/report.md`)로 작성한다.
2. GitHub에는 리포트 파일을 push하지 않는다. 텔레그램으로만 전송한다 (환경변수 `TG_BOT_TOKEN`, `TG_CHAT_ID`는 이미 설정되어 있음):
   ```
   python3 scripts/send_telegram.py /tmp/report.md
   ```
3. 상태를 기록한다 (환경변수 `GH_TOKEN`, `GH_REPO`는 이미 설정되어 있음):
   ```
   python3 scripts/update_status.py korea-noon <YYYY-MM-DD> success
   ```
   실패한 단계가 있으면 `success` 대신 `failed`로 기록한다.
